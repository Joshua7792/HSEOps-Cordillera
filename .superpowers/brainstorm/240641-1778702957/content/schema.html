Key design decisions

  • source field on CertEntry — values: pdf_import, manual, excel_migration. You can always see where a date came from.
  • Newest date wins — when a PDF import brings a date for a cert a worker already has, it only overwrites if the new date is more recent. Existing good data is never silently deleted.
  • Cascade deletes — deleting a Worker removes their CertEntries. Deleting a Contractor doesn't auto-delete Workers (safety: you'd need to reassign first).
  • Status computed at runtime — green/yellow/red is never stored in the DB. It's calculated from completed_on + today's date, exactly like today's excel_reader.py does it. No staleness risk.
  • Alembic for migrations — schema changes are versioned. If we add a field later, alembic upgrade head handles it without touching your data.
