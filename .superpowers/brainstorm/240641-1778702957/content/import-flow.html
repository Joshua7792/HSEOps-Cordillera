Design — Section 3: PDF Import Flow + CRUD

How a PDF goes from your folder into the dashboard, and how you edit records after.

Import flow — step by step

1
You drag one or more Anejo 3 PDFs onto the Import page (or click to browse). Multiple files upload in one go.
2
POST /api/import/pdf receives the file. The existing import_pdf.py parser runs — same Spanish-locale date logic, same name normalization, same Anejo 3 field extraction. Nothing changes in the parser.
3
Results write to SQLite — new workers/contractors created, cert dates upserted (newest date wins). An ImportBatch record is saved with the full warning list.
4
Response returns immediately: records added, records updated, records skipped, and every warning (e.g. "Worker 'Juan Rosario' matched to contractor by fuzzy name — verify", "Date field missing for OSHA 10").
5
Dashboard updates instantly. Navigate to Overview, Heatmap, or Action Center — data is already there. No Refresh button, no file-lock wait.

CRUD endpoints added

PUT /api/workers/{id} — edit name, status, email, job title, etc.
PUT /api/contractors/{id} — edit contact, specialty, notes
POST /api/contractors — add contractor manually before first PDF
PUT /api/certs/{id} — edit cert category or validity
POST /api/certs — add cert type manually
PUT /api/cert-entries/{id} — fix a wrong date
DELETE /api/cert-entries/{id} — remove a cert entry entirely
GET /api/import/history — list of all past PDF imports + warnings
