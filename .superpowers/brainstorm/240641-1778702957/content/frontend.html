What doesn't change in the frontend

  • All 6 existing pages — same layout, same data, same TypeScript types
  • DashboardContext — still fetches from /api/excel/dashboard (same URL)
  • Status color logic — green/yellow/orange/red, same thresholds
  • i18n EN/ES translations — no new strings needed for core pages
  • Heatmap, Action Center filters, KPI strip — all identical
  • Theme (light/dark) — unchanged

One UX improvement: replace the Refresh button

The current Refresh bar exists because Excel could change on disk at any time. With SQLite, data only changes when you do something (import or edit). The RefreshBar stays but becomes a lightweight "Last updated X minutes ago" indicator with a manual refresh option — no longer a required step before trusting the data.
